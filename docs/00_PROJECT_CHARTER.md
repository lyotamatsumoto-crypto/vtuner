# 00_PROJECT_CHARTER.md

## Purpose

VTuner は、配信画面の端に常駐する「仲介者キャラクター」を提供するプロジェクトである。  
このキャラクターは、配信者本人でも単なるコメント読み上げ機械でもなく、配信者と視聴者の間に立つ存在として振る舞う。

VTuner の主な目的は次のとおり。

- 配信中のコメント反応を整理し、必要に応じて仲介者らしい形で伝える
- 配信者の補助として、質問・反応・場の変化をわかりやすく示す
- 視聴者とのやり取りに、キャラクター性とチャンネル固有の文化を持たせる
- ユーザーが見た目・性格・口調・反応を調律できるようにする
- 配信後レビューを通じて、仲介者を育てられる構造を持つ

本プロジェクトは、単なるコメント読み上げツールではなく、  
**「カスタム可能な仲介キャラクター付き配信UI」** を目指す。

---

## Product Definition

VTuner は、次の性質を持つ。

- 配信画面の端に表示される 2D キャラクター
- コメントや配信状況に応じて反応する仲介者
- 配信者向けと視聴者向けで異なる向き・発話を持てる存在
- 見た目、位置、口調、性格、反応ルールを調整できるツール
- 配信後レビューにより、見逃しコメントや新しい反応を追加できるツール

---

## Core Principles

### 1. 配信中は安定性を優先する
配信中の挙動は、自由生成よりも安定したルールベースを優先する。  
誤判定して不自然な発話を行うより、黙ることを選ぶ設計を基本とする。

### 2. AIは補助役に限定する
AI は配信中のコア挙動を直接担うのではなく、主に以下の補助に用いる。

- 詳細ルール追加用 JSON の生成補助
- 性格タイプ・口調タイプ・語尾タイプ・反応カテゴリなどの定義追加補助
- テンプレートや反応文の下書き生成
- 配信後レビューからの追加候補や patch 候補生成補助

### 3. ユーザーが調律できることを重視する
VTuner は完成済みキャラクターを固定提供するだけでなく、  
ユーザーが自分の配信文化に合わせて調整・改変・育成できることを価値とする。

### 4. JSONは内部契約、UIは人間向けにする
内部では設定データを JSON 系で管理するが、ユーザーが直接 JSON を主操作する設計にはしない。  
主な編集体験はフォームUI、カードUI、プレビュー、レビューで提供する。

### 5. 自動学習ではなく、レビューによる明示的な改善を優先する
配信後の見直しでは、見逃しや未分類コメントをユーザーが意味ごとに仕分けし、  
必要なものだけを明示的に追加・修正する。  
仲介者が勝手に性格や辞書を変えていく設計にはしない。

### 6. 本仕様を育てる前提で進める
本プロジェクトは、短期的なMVPだけを目的とせず、  
本仕様として継続開発しやすい文書基盤・構造を先に整える。

### 7. 共通設定と条件別設定を分ける
キャラ全体に共通する設定と、条件ごとに変わる発話ルールを明確に分離する。  
これにより、UIの破綻や設定の混線を防ぐ。

## In Scope

現時点で本仕様の対象に含めるものは次のとおり。

- 2Dキャラクターの表示
- front / side の表示切り替え
- side 画像の左右反転利用
- キャラクター画像の読み込み（PNG / JPEG）
- プレビュー上でのドラッグ移動と位置調整
- 性格・話し方・反応ルールの設定
- 視聴者向け / 配信者向けの発話分岐
- コメントカテゴリのルールベース分類
- テスト画面でのコメント入力と疑似イベント発火
- 音声の基本設定
- 吹き出しや文字表示の基本設定
- 詳細ルールの追加・編集
- ブラウザAI向けの設定生成補助導線
- AIが返した JSON の取り込み
- 配信後レビュー
- unknown コメントの回収と追加候補化
- patch 方式による設定拡張
- 将来の拡張を見据えたイベント設計・設定設計

---

## Out of Scope

現時点では以下を本仕様の中心対象にしない。

- 配信中の高度な自由生成会話
- ローカルLLM常駐を前提としたリアルタイム制御
- 完全自動学習による性格変化
- 全コメントの完全理解
- 画像アニメーションの高度な補間や複雑なモーション
- back 画像を前提とした初期設計
- 高度なクラスタリングによるレビュー自動整理
- 複数クライアントへの完全同期
- 複雑な課金・マネタイズ設計
- 条件ごとに吹き出し見た目を切り替えること
- フォントや色設定の JSON 拡張を初期段階で主対象にすること

## Initial UX Direction

VTuner の体験は、次の **4画面** を中心に組み立てる。

### 1. プレビュー / テスト
- 配信風の画面内でキャラクターを確認する
- キャラクター位置をドラッグで調整する
- コメントを手入力する
- サンプルコメントや疑似イベントを流す
- 音声や反応を試す
- 画像の仮差し替えを行う
- 吹き出しサイズを実寸で調整する

### 2. 基本設定
- VTuner名、一人称、呼び方を設定する
- キャラプロフィールを設定する
- 性格タイプ、口調タイプ、語尾タイプなどを選ぶ
- 音声、画像、吹き出し、フォント、色などの共通設定を決める
- 反応カテゴリの大枠を有効 / 無効で決める

### 3. レビュー
- 配信後コメントを見返す
- 読み上げた / 未分類 / 無視などの状態を確認する
- 既存カテゴリへの追加、ignore、新規候補化を行う
- patch 候補を整理する

### 4. 詳細設定
- いつ、誰に、何を話すかを決める
- 条件イベント、特定ユーザーイベント、名前イベントなどを編集する
- 候補文を編集する
- 新しいルールを追加する
- 新しい性格タイプ、口調タイプ、語尾タイプ、反応カテゴリ定義を追加する
- ブラウザAI用指示文や JSON 読込を扱う

---

## Configuration Philosophy

設定は単一巨大ファイルではなく、責務ごとに分割する。

初期想定の主要ブロックは次のとおり。

- basic settings
- definitions
- rules
- appearance
- voice
- review / patches

これらは内部契約として保持し、  
ユーザーにはフォームUI、カードUI、プレビューを通じて編集させる。

---

## Basic Settings vs Detailed Rules

VTuner では、次の分離を強く守る。

### Basic Settings に置くもの
キャラ全体に共通して効くもの。

- VTuner名
- 一人称
- 視聴者 / 配信者の呼び方
- キャラプロフィール
- 性格タイプ
- 口調タイプ
- 語尾タイプ
- 音声
- 画像
- 吹き出し
- フォント
- 文字色
- 背景色
- 反応カテゴリの大枠

### Detailed Rules に置くもの
条件ごと・ルールごとに変わるもの。

- どの条件で発話するか
- 誰に向けて話すか
- 候補文は何か
- 特定ユーザーへの対応
- 初見対応
- 時間イベント
- 名前イベント
- 返答の例外処理
- 新しい定義の追加

この分離により、設定の役割を明確にし、UIの肥大化を防ぐ。

---

## AI Usage Policy

AI 利用は次の原則に従う。

### 採用する用途
- 詳細ルール追加の補助
- 性格タイプや反応カテゴリなどの追加定義補助
- personality / behavior / templates 系 JSON の生成補助
- 追加カテゴリ候補や追加テンプレート候補の下書き
- 配信後レビュー結果からの patch 候補生成補助

### 採用しない用途
- 配信中の主要発話決定
- 常時AI依存のコメント処理
- ユーザーが制御しづらい自由会話の中核

---

## Safety and Control

- 不確実な分類は `unknown` として扱う
- 無理に喋らせず、黙ることを許容する
- 名前読み上げは ON/OFF および制限設定を持つ
- 長い名前、記号だらけの名前、NG名、センシティブ語を含む名前には制御を入れる
- チャンネル固有の荒い口調や軽口は、デフォルトではなくユーザー設定前提とする
- Detailed Rules で追加した内容も、最終的にはユーザーが確認・採用する構造を優先する

---

## Growth Loop

VTuner は、配信中に完璧であることよりも、  
配信後のレビューを通じて改善していける構造を重視する。

基本ループは次のとおり。

1. 配信中はルールベースで安定動作する
2. 未分類・未採用コメントを記録する
3. 配信後レビューで意味を仕分ける
4. 必要なものだけを設定や patch として追加する
5. 次回配信で改善された状態を試す

---

## Non-Goals for Early Implementation

初期段階では、次のことを急がない。

- すべてのカテゴリの網羅
- 音声の高度制御
- back 表示の実装
- 複雑なアニメーション
- マネタイズ仕様の確定
- 完全な本番運用レベルの長期耐障害性
- 条件ごとの見た目切り替え
- 無限に自由なUI拡張
